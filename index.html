About

I got into programming the way I wish more people did — by shipping something. In high school, I built and published a game on Steam. Not a prototype, not a class project: a real game, rated Very Positive by players who found it on their own. That experience set the tone for everything that followed.

I moved through undergrad in 2.5 years with a 4.0, not because I was racing, but because I was always chasing the next thing I wanted to build. Now I'm a first-semester MSE student in CSE at the University of Michigan, where I also teach as a GSI for EECS 281 (Data Structures & Algorithms) and EECS 291 (Intro to Applied Data Structures & Algorithms).

These days my focus has shifted to AI systems — specifically, what it looks like when agents stop being toys and start doing real work. I'm building a multi-agent orchestration platform that's running 24/7 on a Mac Mini in my room, handling programming tasks, research, writing, and code review autonomously. It's less of a side project and more of an experiment in what one person can operate with the right infrastructure.

This summer I'm at Microsoft on Azure hyperscale networking — the layer of infrastructure most people never think about until it breaks.

Outside of all that, I play Rocket League on the UMich varsity esports team. It's the one thing that keeps me honest about how long mastery actually takes.

Projects

PAW Engineering

Personal AI Workforce Platform

What happens when you give one person a full engineering team made of AI? PAW is a multi-agent orchestration platform where specialized autonomous agents — programmer, researcher, writer, code reviewer, architect — work in parallel on real tasks, coordinated by an always-on orchestrator. The system handles its own task queues, model selection across Claude/GPT/Codex, circuit breaking, failure recovery, and deduplication. It runs 24/7 on a Mac Mini, processing tasks while I sleep. Workers operate in isolated workspaces with shared semantic memory, and the orchestrator automatically selects the right model tier (from local models to Opus) based on task complexity.

5 specialized agent types 24/7 autonomous operation Tier-based model routing Circuit breaking & failure recovery Semantic memory across agents

Lobs

Always-On Personal AI Agent

My personal AI executive assistant — not a chatbot, but an autonomous agent that runs continuously. Lobs manages the PAW workforce, creates and triages tasks, handles communications across Discord and other platforms, sets reminders, runs scheduled health checks, and proactively identifies work that needs doing. It has persistent semantic memory (vector search over daily logs and long-term knowledge), scheduled 30-minute heartbeats for system monitoring, and a web dashboard (Nexus) for real-time visibility into everything happening across the agent fleet. The goal: make it feel like having a competent colleague who's always working in the background.

Persistent semantic memory 30-min autonomous heartbeats Multi-platform messaging Real-time Nexus dashboard Proactive task creation
