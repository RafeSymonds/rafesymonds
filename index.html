About

From publishing games to building AI workforces.

I started programming to ship a game to Steam in high school. That curiosity for building real, shipped things never left — it just evolved.

Now I'm an MSE student in Computer Science at the University of Michigan, where I also teach as a GSI for EECS 281 (Data Structures & Algorithms) and EECS 291 (Intro to Applied Data Structures & Algorithms). This summer I'm joining Microsoft's Azure hyperscale networking team.

My focus has shifted to AI systems — specifically, building agents that do real, autonomous work. Not chatbots. Not demos. Agents that run continuously, coordinate with each other, and actually get things done.

Flagship · AI Infrastructure

PAW Engineering

A multi-agent orchestration platform running 24/7 on a Mac Mini. Five specialized agents — programmer, researcher, writer, reviewer, architect — work autonomously in parallel, coordinated by an always-on orchestrator with task queues, model routing (Claude/GPT/Codex), circuit breaking, failure recovery, and deduplication.

  • Isolated workspaces with shared semantic memory
  • Tier-based model selection (local → Opus)
  • 5 concurrent agent types working in parallel
paw-engineering.com GitHub
Flagship · Personal AI Agent

Lobs

Always-on personal AI agent — not a chatbot. Lobs manages my PAW workforce, creates and triages tasks, handles communications across Discord, and runs 30-minute heartbeat cycles to proactively identify what needs doing. Persistent semantic memory via vector search keeps context across every session.

  • Persistent semantic memory with vector search
  • 30-min heartbeats, proactive task creation
  • Nexus web dashboard for real-time visibility
